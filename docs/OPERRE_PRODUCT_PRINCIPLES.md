# Operre Product Principles

## Product identity

- Product name: Operre
- Domain: operre.com

Operre is a lightweight, independent, cross-platform project working environment.

The domain was originally inspired by operation research. The name also carries useful meaning around operation, operate, organize, prepare, execute, repository, workspace, and the German word "Oper" as a stage/performance metaphor.

Internal meaning:

> Operre is a lightweight place where projects are prepared, organized, and performed.

The stage/performance metaphor must remain subtle. Operre must stay technical, clean, and lightweight.

## Core product direction

Operre starts like a classic old Windows 10 Notepad-style application:

- opens quickly;
- allows immediate `.txt` writing;
- saves files simply;
- does not require cloud;
- does not require an account;
- does not require Ideboard;
- does not require GitHub;
- does not require AI.

Windows 11 Notepad is not the reference experience. The older simple Notepad feeling is the preferred lightweight reference.

## Growth model

Operre core must stay small.

Operre grows through:

- Built-in Features;
- Extensions;
- Connectors;
- Services.

The product must not start as a huge IDE.

## Independence

Operre is independent.

It must not be:

- an Ideboard shell;
- a GitHub client shell;
- an AI client shell;
- a cloud-first editor;
- a VS Code clone;
- a Cursor clone.

Ideboard is a separate application and separate project. Ideboard may later connect to Operre through an optional Connector Extension.

## Source and distribution model

Operre is not planned as open source.

Rules:

- development repository should normally be private;
- current public visibility was temporary for inspection/setup;
- basic/core application may be free;
- future extensions may be free or paid;
- a separate public repository may later be created for public docs/distribution/user-facing materials.

## Core philosophy

> Core-first. Cloud-optional. Extension-driven. Privacy-first.

## Parallel project rule

Operre can be developed in parallel with LogisticSearch.

LogisticSearch remains a long-running major project. Operre must not weaken LogisticSearch discipline.

Operre should reuse the same seriousness:

- audit first;
- document decisions;
- update TODO;
- commit;
- push;
- post-push audit.

## Source 2 product baseline

The `2.odt` review confirms the product baseline:

- Operre is a very lightweight project platform.
- Operre starts like old Windows 10 Notepad.
- The first value is immediate `.txt` writing.
- Operre is not open source.
- Basic/core may be free.
- Extensions may be paid later.
- Ideboard remains separate.
- GitHub remains optional.
- LogisticSearch can continue in parallel.
